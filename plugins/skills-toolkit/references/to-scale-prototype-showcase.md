# Pattern: to-scale prototype showcase (design flow)

A design flow that works well for any visual build or design task: make the prototype tangible
before committing.

1. Render a to-scale styled SVG of the design: a dimensioned elevation plus a plan view.
2. Offer a small set of finish or variant swatches, each drawn to scale and clearly labelled.
3. Converge with a single question asking which direction the user prefers.

Keep everything to scale: true aspect ratio, real element sizes (for example masonry blocks at
their actual mm size), and a scale reference such as a 1 m bar. If a part genuinely cannot be to
scale, label it as not-to-scale rather than presenting it as representative.

Why it works: it surfaces proportion and texture decisions early, in a form the user can react
to directly, and it reads as a clean way to showcase a prototype or pattern. Pair any build that
produces visuals with this flow.
