# Field Mapping - Claude Code to OpenCode

Authoritative mapping tables used by Mode B (Sync) and `scripts/sync_agents.py`. Verified against opencode.ai docs, June 2026. OpenCode releases roughly fortnightly - re-verify these tables before any major migration phase.

## Discovery paths

| Artifact | Claude Code reads | OpenCode reads |
|---|---|---|
| Skills (SKILL.md) | `~/.claude/skills/`, `.claude/skills/`, plugin `skills/` dirs | `.opencode/skills/`, `~/.config/opencode/skills/`, **`.claude/skills/`**, **`~/.claude/skills/`**, `.agents/skills/`, `~/.agents/skills/` |
| Agents | `.claude/agents/`, `~/.claude/agents/`, plugin `agents/` | `.opencode/agent/` (or `agents/`), `~/.config/opencode/agent/` - does NOT read `.claude/agents/` |
| Commands | `.claude/commands/`, `~/.claude/commands/` | `.opencode/command/`, `~/.config/opencode/command/` |
| Rules | `CLAUDE.md` (project), `~/.claude/CLAUDE.md` (global) | `AGENTS.md` first; falls back to `CLAUDE.md`; reads global `~/.claude/CLAUDE.md` unless `OPENCODE_DISABLE_CLAUDE_CODE_PROMPT=1` |
| MCP | `~/.claude.json` / `.mcp.json` `mcpServers` | `opencode.json` / `opencode.jsonc` `mcp` block |

**Consequence**: skills and rules need no translation - they are single-source. But skills that live inside a plugin (`plugins/<name>/skills/`) are NOT in any OpenCode discovery path, so they need EXPOSURE (not generation): add the plugin root to `skills.paths` in `opencode.json`, with a `permission.skill` deny-list for the reference-only ones. Full mechanics in `references/skill-exposure.md`. Agents, commands, and MCP still need translation.

## Skill frontmatter (SKILL.md)

| Field | Claude Code | OpenCode |
|---|---|---|
| `name` | Required. Lowercase letters, numbers, hyphens, max 64, must match folder | Required. Same regex (`^[a-z0-9]+(-[a-z0-9]+)*$`), must match folder |
| `description` | Required. 1024 max per spec; listing truncates at 250 chars, so treat 250 as the working cap | Required. 1-1024 chars, no truncation behaviour documented |
| `allowed-tools` | Honoured (dash-per-line YAML) | **Ignored** |
| `disable-model-invocation` | Honoured (blocks auto-invocation) | **Ignored** - and every skill is also exposed as a `/name` command |
| `version`, `category`, `tags`, `last-updated`, `user-invocable` | Ecosystem conventions, harmless | **Ignored** (unknown fields are dropped without error) |
| `license`, `compatibility`, `metadata` (string map) | `metadata`/`license` tolerated | Recognised |

**The drift trap**: OpenCode ignoring fields is silent. A side-effecting skill that relies on `disable-model-invocation: true` becomes model-invocable on OpenCode. Validate mode flags this (rule TR-04).

## Agent frontmatter

Canonical (Claude-style) in, OpenCode out. Filename becomes the agent name on the OpenCode side (no `name` field there).

| Canonical field | OpenCode field | Transform |
|---|---|---|
| `name` | (filename) | Drop the field, name the file `<name>.md` |
| `description` | `description` | Pass through. REQUIRED - missing means E4, skip file |
| `tools` (allowlist: list or comma string) | `tools` (boolean map) | Lowercase each via the tool map in `references/tool-translation.md`; unknown or Claude-only names dropped with a warning. `tools` map is deprecated on OpenCode in favour of `permission`, but still works - prefer it for generation simplicity |
| `model` (`sonnet`, `claude-sonnet-4-x`...) | `model` (`provider/model-id`) | Map via `config.model_map`; unmapped means E5: emit `anthropic/<original>` plus `TODO-VERIFY` |
| `temperature` | `temperature` (0-1) | Pass through if present |
| (none) | `mode` (`primary` / `subagent` / `all`) | Inject `config.default_agent_mode` (default `subagent`) |
| `when_to_use`, `version`, `author`, `created`, `updated`, `category`, `tags` | (none) | Strip from frontmatter; the body keeps its When to Use section |
| (none) | `disable`, `hidden`, `permission` | Not generated by default; add per-agent overrides in config if needed |

### Worked example

Canonical `.claude/agents/content-brainstormer.md`:

```yaml
---
name: content-brainstormer
description: Content creation workflow orchestrator. Entry point coordinating research and style agents.
tools:
  - WebSearch
  - Read
  - Write
  - AskUserQuestion
  - TaskCreate
model: sonnet
---
```

Generated `.opencode/agent/content-brainstormer.md`:

```yaml
---
# GENERATED by opencode-sync from .claude/agents/content-brainstormer.md - edit the canonical source
description: Content creation workflow orchestrator. Entry point coordinating research and style agents.
mode: subagent
model: anthropic/claude-sonnet-4-5
tools:
  websearch: true
  read: true
  write: true
  question: true
---
```

Note `AskUserQuestion` mapped to `question` and `TaskCreate` dropped with a warning - the body must already carry the degradation wording (rule OC-05).

## Model map

Defaults shipped in config - ALWAYS verify against the live models.dev registry before trusting them; ids change:

| Canonical | OpenCode |
|---|---|
| `sonnet` / `claude-sonnet-4-x` | `anthropic/claude-sonnet-4-5` |
| `opus` / `claude-opus-4-x` | `anthropic/claude-opus-4-1` |
| `haiku` / `claude-haiku-4-x` | `anthropic/claude-haiku-4-5` |
| anything else | `anthropic/<original>` + `TODO-VERIFY` (E5) |

When OpenCode runs on a non-Anthropic provider (Copilot, Zen, local), the map should point at that provider's ids instead - the map is per-repo config, not a constant.

## Command frontmatter

| Claude Code | OpenCode | Note |
|---|---|---|
| `description` | `description` | Pass through |
| `allowed-tools` | (none) | Drop |
| `argument-hint` | (none) | Drop; `$ARGUMENTS`, `$1`..`$9` placeholders work identically in the body |
| `model` | `model` | Apply model map |
| (none) | `agent`, `subtask` | Optional additions on the OpenCode side |

Reminder: the ecosystem rule is skills-only. OpenCode exposes every skill as `/name` anyway, so generate commands only for genuine legacy stragglers.

## MCP config

Claude Code (`~/.claude.json` or `.mcp.json`):

```json
{ "mcpServers": { "slack": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-slack"] } } }
```

OpenCode (`opencode.json`):

```json
{
  "mcp": {
    "slack": {
      "type": "local",
      "command": ["npx", "-y", "@modelcontextprotocol/server-slack"],
      "enabled": true
    }
  }
}
```

Differences that bite: OpenCode needs `type` (`local` or `remote`), takes `command` as a single array (no separate `args`), and requires `enabled`. Remote servers use `{"type": "remote", "url": "...", "enabled": true}`. Tool name prefixes differ per client, so skill bodies should reference MCP tools by intent ("the Slack search tool") wherever a body is shared.

### Remote MCP OAuth (the undocumented fields that bite)

OpenCode auto-handles OAuth for remote MCP servers via Dynamic Client Registration (RFC 7591). When the auth server does not support DCR (for example Slack's hosted MCP at `mcp.slack.com`), it fails with `Incompatible auth server: does not support dynamic client registration`. To use a pre-registered client instead, the `oauth` object accepts:

- `clientId`, `clientSecret`, `scope` - documented.
- `redirectUri`, `callbackPort` - real but UNDOCUMENTED (verified in the binary, OpenCode 1.17). There is NO `callbackUri` field; setting `callbackUri` is silently ignored and wastes hours. Redirect precedence is:

  ```
  redirectUri ?? `http://127.0.0.1:${callbackPort ?? 19876}/mcp/oauth/callback`
  ```

`callbackPort` alone only changes the port (host stays `127.0.0.1`, path stays `/mcp/oauth/callback`); use `redirectUri` for a full override. The value is sent verbatim as the OAuth `redirect_uri` AND used to bind the local catcher, so it must byte-match the URI registered in the provider's app - Slack does exact-string matching, so `http://localhost:3118/callback` does NOT equal `http://127.0.0.1:3118/callback`. Working Slack block (no app-owner change needed when the localhost callback is already registered on the app):

```json
"slack": {
  "type": "remote",
  "url": "https://mcp.slack.com/mcp",
  "enabled": true,
  "oauth": {
    "clientId": "<your-registered-client-id>",
    "callbackPort": 3118,
    "redirectUri": "http://localhost:3118/callback"
  }
}
```

## Rules files

- Canonical: `AGENTS.md`. Claude Code compatibility: `ln -s AGENTS.md CLAUDE.md` (commit the symlink; on Windows ensure `core.symlinks=true`).
- Precedence on OpenCode: project `AGENTS.md` beats project `CLAUDE.md`; global `~/.config/opencode/AGENTS.md` beats the `~/.claude/CLAUDE.md` fallback.
- Claude-specific `@import` lines and path-scoped CLAUDE.md tricks do not translate - keep AGENTS.md plain markdown.
- Auth context (April 2026 onward): subscription OAuth is Claude-Code-only by Anthropic ToS; OpenCode runs on API keys, Copilot, OpenCode Zen, or local models. Plan token budgets accordingly.
